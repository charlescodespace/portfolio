# Portfolio — Charles S

This folder contains a static personal portfolio for Charles S showcasing projects, skills, certifications, and education. The site is authored with plain HTML, CSS and vanilla JavaScript with small animation and 3D enhancements.

Live preview
- Open `portfolio/index.html` in a browser via a local HTTP server (recommended) or deployment host.

Quick start (local server)
- Python (quick):

```powershell
# from the repository root (where `portfolio` is located)
python -m http.server 8000
# then open http://localhost:8000/portfolio/index.html
```

- Node (http-server):

```powershell
npx http-server -p 8000
# then open http://localhost:8000/portfolio/index.html
```

Why use a server
- Opening the page directly with the `file://` protocol causes browser security restrictions (module imports, CORS, and resource loading). Serving the folder over HTTP resolves console errors such as unsafe file:// loads and enables ES module imports.

What you'll find here
- `index.html` — main single-page portfolio.
- `style.css` — site styling and component rules.
- `script.js` — interactive behavior, GSAP animations, typewriter, and Three.js-powered background canvases.
- `project-modal.js` — project modal helpers (if present).
- `assests/` — images and project thumbnails used across the site.

Key features
- Hero section with animated typewriter and profile snapshot.
- Project gallery with modal that extracts project details from each card.
- Skills grid with interactive tiles and skill detail modal.
- Lightweight 3D background and product/network canvases built with Three.js.
- GSAP/ScrollTrigger animations for reveal effects.

Primary technologies
- HTML, CSS, JavaScript (vanilla)
- Three.js (ES Module build) for 3D canvases
- GSAP + ScrollTrigger for animations
- Devicons for skill icons

How to edit content
- Projects: each project card is an `.project-card` in `index.html`. Update the `project-image`, `project-tag`, `h3` title, `.location`, and `.description-block` sections. The project modal reads these blocks automatically.
- Skills: the skills live under the `#skills` section in `index.html`. There are two tile styles used in the page: simple `.skill-tile` and enhanced `.skill-badge` (icon + name + level). Keep the `img` and `span` or the `skill-badge` markup consistent when adding new skills.
- Assets: place new images under `assests/` and reference them with relative paths (e.g. `assests/my_image.jpg`).

Troubleshooting
- Console warns about three.js deprecated builds: the page uses the ES module build of Three.js and assigns it to `window.THREE`. If you still see warnings, ensure you open the page over `http://` (see Quick start).
- If skill/project modals don't populate: ensure the HTML structure for `.project-card` and `.skill-tile` follows the expected pattern (images, headings, `.description-block` headings).

Deployment suggestions
- Static hosts like GitHub Pages, Netlify, or Vercel work well. Deploy the repository root or the `portfolio/` subfolder as a static site.
